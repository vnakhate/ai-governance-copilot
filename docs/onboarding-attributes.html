AI tool onboarding · attribute matrix

What an onboarding request can capture

~55 attributes across 9 categories. Which are required depends on the risk tier (data class + use) and whether the tool is vendor or internal. now = in the demo schema; gap = to add.

Tier 1 · baseline

Every request. Identity, use, data class, residency, training-exclusion, DPA, certs, approval.

Tier 2 · sensitive

Data ≥ Confidential or risk ≥ limited. + sub-processors, encryption, access, DPIA, incident response.

Tier 3 · high-risk

Regulated-PII or high-risk use. + model provenance, training data, bias, explainability, oversight.
