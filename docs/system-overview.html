02 Architecture decisions

What got locked in review

1A Verdict in code

The risk verdict is computed in the endpoint as a pure assess() function, not LLM reasoning. Glean needs a hosted endpoint anyway, so the verdict rides along — deterministic and unit-testable.

2A Profiles in code, policy indexed

Tool profiles drive the verdict; the policy corpus is indexed into Glean so the agent cites real clauses. That's the honest answer to “what is Glean doing here?”

4A Two gated actions

/assess is read-only; /capture writes to Vanta and fires only after an explicit “Yes”.

5A Single source of truth

Both actions call the same assess(), so the captured record is provably identical to the verdict shown.

3B ngrok hosting

Tunnel to localhost for fast iteration. Mitigations: reserved domain, recorded backup clip, phone hotspot.

Dual Chat + headless

Chat copilot for the demo; the same /assess+/capture core is callable headlessly by procurement / ITSM / a script.
